Introduction

Conventions is an integrator framework of Java EE technologies with the objective to easy the development of enterprise rich web applications. The framework leverages other frameworks such as Hibernate, Primefaces and CODI to delivery solutions to common problems like exception handling, true pagination, helper UI components, internationalization, annotated queries etc...

This guide is divided into four parts organized by Conventions archctectural layers plus one part for general utilities components provided by the framework and one more part for Conventions Scaffolding a code generation tool based on JBoss Forge. Part I, “The controller layer” is represented by conventions managed beans which are wrappers to CDI beans; Part II, “The service layer” shows how the framework is integrated with CDI, Hibernate and EJB to provide database operations; Part III, “The view layer” discusses the view layer and its UI helper components; Part IV, “The model layer” talks about the model layer and its helper classes; Part V, “General utilities” walks throught utilities components such as properties provider, security helper, anotated queries, i18n, paginator and so on; Part VI, “Scaffolding” Demostrates Conventions Scaffold, a tool based on JBoss Forge to enable boilerplate code generation for Conventions Framework.

Chapter 1. Getting Started

Getting started with Weld Extensions is easy. If you are using Maven, then you can declare a dependency on Weld Extensions (org.jboss.weld:weld-extensions:${weld.extensions.version}, make sure you have the JBoss Maven repository enabled). Otherwise, add the jar to your compile time and runtime classpath.

Most of Weld Extensions has very few dependencies:

  • javax.enterprise:cdi-api
  • org.slf4j:slf4j-api
  • org.jboss.logging:jboss-logging-api
  • javax.el:el-api
  • javax.inject:javax.inject
  • javax.transaction:jta

Tip

The POM for Weld Extensions specifies the versions required. If you are using Maven 3, you can easily import the dependencyManagement into your POM by declaring the following in your depdendencyManagement section: 


         <dependency>

            <groupId>org.jboss.weld</groupId>

            <artifactId>weld-extensions</artifactId>

            <version>${weld.extensions.version}</version>

            <type>pom</type>

            <scope>import</scope>

         </dependency>

Some features of Weld Extensions require additional dependencies (which are declared optional, so will not be added as transitive dependencies):

org.javassist:javassist
Service Handlers, Unwrapping Producer Methods
javax.servlet:servlet-api
Accessing resources from the Servlet Context
